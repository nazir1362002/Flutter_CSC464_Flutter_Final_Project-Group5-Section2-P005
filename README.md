# Student Attendance & Class Routine App

A Flutter + Firebase mobile app for teachers to manage courses, students, attendance, and class routines.

---

## Features

- **Course Management** — Add, edit, delete courses
- **Student Enrollment** — Add/edit/remove students per course
- **Attendance** — Mark Present/Absent, save per date, view full history
- **Attendance Summary** — Total classes attended + percentage per student
- **Class Routine** — Add weekly schedule with day, time, and course

---

## Firebase Firestore Structure

```
/courses
  /{courseId}
    name: string
    code: string

/courses/{courseId}/students
  /{studentId}
    name: string
    studentId: string

/courses/{courseId}/attendance
  /{attendanceId}
    date: timestamp
    records: {
      studentDocId1: "Present",
      studentDocId2: "Absent"
    }

/routine
  /{routineId}
    courseId: string
    day: string
    time: string
```

## Setup Instructions

### Prerequisites

- Flutter SDK 3.x installed ([flutter.dev](https://flutter.dev))
- Firebase account ([console.firebase.google.com](https://console.firebase.google.com))
- Dart SDK 3.x (comes with Flutter)

---

### Step 1 — Install dependencies

```bash
flutter pub get
```

---

### Step 2 — Create a Firebase project

1. Go to [Firebase Console](https://console.firebase.google.com)
2. Click **Add Project** and follow the steps
3. In your project, go to **Firestore Database** → **Create database**
4. Choose **Start in test mode** (for development), then select your region

---

### Step 3 — Connect Firebase to Flutter

Install the FlutterFire CLI:

```bash
dart pub global activate flutterfire_cli
```

Then run in the project root:

```bash
flutterfire configure
```

This command will:
- Ask you to select your Firebase project
- Auto-generate `lib/firebase_options.dart` with real credentials
- Download `google-services.json` for Android
- Download `GoogleService-Info.plist` for iOS

> **Important:** The `lib/firebase_options.dart` file included in this project is a placeholder. `flutterfire configure` will replace it with real credentials.

---

### Step 4 — Android Setup

1. Open `android/app/build.gradle`
2. Make sure the `applicationId` matches what you registered in Firebase
3. The `google-services.json` (generated in Step 3) goes in `android/app/`

---

### Step 5 — iOS Setup

1. Open `ios/Runner.xcworkspace` in Xcode
2. The `GoogleService-Info.plist` (generated in Step 3) goes in `ios/Runner/`
3. Make sure the Bundle Identifier matches Firebase

---

### Step 6 — Run the app

```bash
# Run on Android emulator or device
flutter run

# Run on iOS simulator
flutter run -d ios

# Run on a specific device
flutter devices        # list connected devices
flutter run -d <id>    # run on that device
```

## Firestore Security Rules (Recommended)

After testing, update your Firestore rules in Firebase Console → Firestore → Rules:

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /{document=**} {
      allow read, write: if request.time < timestamp.date(2025, 12, 31);
    }
  }
}
```

For production, add authentication to restrict access.

## Troubleshooting

**`firebase_options.dart` errors** → Run `flutterfire configure` to regenerate the file.

**Firestore permission denied** → Make sure Firestore is created and rules allow read/write.

**`google-services.json` not found** → Run `flutterfire configure` again, ensure the file is in `android/app/`.

**Fonts not loading** → Run `flutter pub get` then `flutter clean && flutter run`.
